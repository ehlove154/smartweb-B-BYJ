form
-form (양식) 태그는 서버로 전송하고자 하는 사용자 입력 값을 받는 입력태그들 (input, select, textarea 등)을 자식으로 가지기 위한 태그
-자동X
-시맨틱O
-하나의 양식 태그는 서로 유관한 입력 태그만 가지는 것이 좋다.
-양식 태그는 서로 선조, 부모, 자식, 자손 관계를 형성하여서는 안된다.
-속성
	*method: 요청을 전송할 방식 지정
	  get: GET 방식으로 지정
	  post: POST 방식으로 지정
	*name: 양식 태그의 이름 지정(JS에서 document.forms 멤버의 키 값으로 접근 가능)
	*novalidate: 양식 태그의 제출(submit)이벤트 발생시 이가 가지는 입력 태그들이 요구하는 요구사항을 무시하게 한다. 속성 값을 가지지 않음

input
-input태그는 사용자로부터 입력값을 받기 위해 사용하는 태그
-자동O
-시맨틱O
-속성
	*name: 가지고 있는 값을 서버로 전송할때 사용하는 식별자
	*type: 해당 입력 태그의 종류를 지정
	   text: 별다른 특성이 없는 일반 텍스트
	   button: 별다른 특성이 없는 일반 버튼. 표시할 텍스트는 value값으로 설정한다.
	   checkbox: 여부(참, 거짓)값을 가지기 위해 사용
	   color: 색상을 선택하기 위해 사용, 데이터 전송시 HEX컬러 문자열로 전송
	   date: 날짜를 선택하기 위해 사용
	   datetime-local: 날짜와 시간을 선택하기 위해 사용
	   time: 시간을 선택하기 위해 사용
	   email: email을 입력받기 위해 사용
	   file: file을 입력받기 위해 사용. 주로 첨부파일 기능을 구현하기 위해 사용. 양식 태그를 통해 파일을 전송하려면 아래 요건을 모두 만족하여야 한다.
		*양식 태그의 enctype 속성 값이 multipart/form-data여야 한다.
		*양식 태그의 method 속성 값이 post여야 한다.
	   hidden: 숨겨진 입력 태그를 위해 사용. 주로, 서버로 전송은 되어야 하나 사용자가 직접 입력하는 값이 아닌(보안상의 이유로 대조되는 salt값 등) 것들을 포함하기 위해 사용.
	   number: 숫자값을 입력받기 위해 사용
	   password: 비밀번호를 입력받기 위해 사용
	   radio: 여러 선택지 중 하나를 선택할 수 있게하기 위해 사용. name 속성 값이 같은 라디오 중 하나만 선택할 수 있다.
	   range: 범위 내의 숫자를 선택하기 위해 사용하는 바(bar)
	   reset: 속해있는 양식 태그가 가지고 있는 입력 태그들의 값을 초기화하는 버튼. 표시할 텍스트는 value값으로 설정
	   search: 검색키워드를 입력받기 위해 사용
	   submit: 속해있는 양식 태그를 제출하기 위한 버튼. 표시할 텍스트는 value속성 값으로 설정
	   tel: 연락처를 입력받기 위해 사용
	*autofocus: 페이지가 로드되었을 때 해당 입력 태그에 자동으로 포커스를 줄 것인가에 대한 속성. 속성 값을 가지지 않는다.
	*checked: type 속성 값이 checkbox 혹은 radio 일때 체크 된 상태로 설정 할 것인가에 대한 속성. 속성 값을 가지지 않는다.
	*disabled: 비활성화 할 것인가에 대한 속성. 속성 값을 가지지 않는다.
	*max: type속성 값이 number, range 등 숫자와 관련이 있을 때 가질 수 있는 최댓값.
	*min: type속성 값이 number, range 등 숫자와 관련이 있을 때 가질 수 있는 최솟값.
	*step: type속성 값이 number 혹은 range일때 한 틱(tick)에 변화 할 단계 값
	*maxlength: 문자열의 최대 길이 (값이 비어있을 때 검사하지 않음)
	*minlength: 문자열의 최소 길이 (값이 비어있을 때 검사하지 않음)
	*multiple: type 속성 값이 file일때 파일을 여러개 선택 할 수 있게 한다. 속성값을 가지지 않는다.
	*placeholder: 값이 비어있을때 보여줄 텍스트를 지정
	*readonly: 읽기 전용으로 지정하기 위해 사용. 속성값을 가지지 않는다.
	*required: 필수 필드이며 값이 비어있을 수 없음을 의미. 속성값을 가지지 않는다.
	*spellcheck: 브라우저에 내장되어 있는 철자 확인 기능을 사용 할 것인가에 대한 설정
	   default: 기본값으로 브라우저의 설정에 따른다.
	   true: 강제 사용
	   false: 강제 미사용
	*pattern: 해당 필드가 가져야할 값에 대한 정규 표현식을 지정한다.
	*accept: type이 file일때 파일 선택 창에서 기본적으로 선택하게 할 파일의 형식을 지정한다. 별도로 지정하지 않으면 모든 파일을 선택할 수 있게 한다. 속성값 지정은 .확장자 (ex>.jpg, .png 등) 혹은 MME 타입(ex> image/png, image/jpeg 등)을 사용하고, MME 타입의 서브타입은 와일드카드를 허용.(image/*, video/* 등의 형식 사용 가능)또한 이러한 가용 형식을 콤마(,)로 구분하여 여러개 사용 할 수 있다. ex> .jpg, .png, .gif, .webp, image/*
	*value: 해당 입력 태그의 (초기) 값을 지정


button
-button을 위한 태그
-자동x
-시맨틱o
-속성
	*disabled: 비활성화 할 것인가에 대한 속성. 속성 값을 가지지 않는다.
	*name: 자바스크립트가 가지는 양식 태그 객체(HTMLFormElement)의 식별자로 사용하기 위해 할당하는 편
	*type: 해당 버튼 태그의 종류를 지정
	   submit: 기본 값으로 속해있는 양식 태그를 제출한다.
	   button: 별다른 특성이 없는 일반 버튼
	   reset: 속해있는 양식 태그가 가지고 있는 입력 태그들의 값을 초기화


select
-드롭다운방식 혹은 리스트 방식의 항목 선택을 위한 입력 태그
-자동x
-시맨틱o
-직접 값을 가지지 않으며, 자식 혹은 자손으로 값을 가지는 option태그를 가진다.
-속성
	*name: 가지고 있는 값을 서버로 전송할 식별자
	*autofocus: 페이지가 로드되었을 때 해당 입력 태그에 자동으로 포커스를 줄 것인가에 대한 속성. 속성 값을 가지지 않는다.
	*disabled: 비활성화 할 것인가에 대한 속성. 속성 값을 가지지 않는다.
	*multiple: 여러개의 option을 선택할 수 있게 한다. 드롭 다운 형식에서 리스트 형식으로 변하게 된다. 속성값을 가지지 않는다.
	*required: 필수 필드이며 값이 비어있을 수 없음을 의미. 속성값을 가지지 않는다.


optgroup
-서로 유관한 옵션태그들을 묶어 내기 위해 사용
-자동x
-시맨틱o
-하나 이상의 option태그가 서로 관련이 있음을 알리는 시맨틱이기는 하나, 선택이 불가능한 항목이고, 시각적 외의 특별한 기능을 가지지는 않는다.
-select 태그의 자식이어야 하고, option 태그를 자식으로 가질 수 있다.
-속성
	*label: 표시할 텍스트 지정
	*disabled: 비활성화한다. 단, 이가 가지고 있는 option 태그들도 함께 비활성화 된다.속성 값을 가지지 않는다.


option
-select태그가 가지는 실질적인 값이 있는 항목 테그
-자동x
-시맨틱o
-속성
	*disabled: 비활성화한다. 속성 값을 가지지 않는다.
	*hidden: 해당 태그를 숨긴다. 속성 값을 가지지 않는다.
	*selected: 해당 태그를 초기에 선택된 상태로 지정. 속성 값을 가지지 않는다.
	*value: 실질적으로 서버에 전송될 값을 지정. 생략할 경우 해당 태그가 가지는 내용으로 대체


textarea
-개행이 가능한 문자열 값을 입력받기 위해 사용
-자동x
-시맨틱o
-초기값을 지정하기 위해서는 input에서 value 속성 값을 지정했던 것과는 달리, 태그의 내부 내용을 작성하면 된다.
-속성
	*name: 가지고 있는 값을 서버로 전송할때 사용하는 식별자
	*autofocus: 페이지가 로드되었을 때 해당 입력 태그에 자동으로 포커스를 줄 것인가에 대한 속성. 속성 값을 가지지 않는다.
	*cols: 초기에 설정될 열의 개수(가로방향으로의 글자수)를 지정. 자연수여야 하며 기본값은 20.
	*rows: 초기에 설정될 행의 개수(세로방향으로의 줄 수)를 지정. 자연수여야 하며 기본값은 2.
	*disabled: 비활성화 할 것인가에 대한 속성. 속성 값을 가지지 않는다.
	*maxlength: 문자열의 최대 길이 (값이 비어있을 때 검사하지 않음)
	*minlength: 문자열의 최소 길이 (값이 비어있을 때 검사하지 않음)
	*placeholder: 값이 비어있을때 보여줄 텍스트를 지정
	*readonly: 읽기 전용으로 지정하기 위해 사용. 속성값을 가지지 않는다.
	*required: 필수 필드이며 값이 비어있을 수 없음을 의미. 속성값을 가지지 않는다.
	*spellcheck: 브라우저에 내장되어 있는 철자 확인 기능을 사용 할 것인가에 대한 설정
	   default: 기본값으로 브라우저의 설정에 따른다.
	   true: 강제 사용
	   false: 강제 미사용


label
-입력 태그와 관련하여 이에 관한 설명을 작성하고 연결시키기 위해 사용
-자동x
-시맨틱o
-연결된 입력 태그에 대한 설명을 문자열로 가지며, 스크린 리더가 연결된 태그에 대한 설명으로 해당 내용을 사용
-label태그를 사용하는 방법은 아래와 같다.
	*label 태그가 직접 입력 태그를 자식 혹은 자손으로 가지게 하는 방법
		<label>
    			<span>이메일</span>
    				<input required maxlength="50" minlength="18" name="email" type="email">
		</label>
	   
	*label 태그의 for 속성 값을 입력 태그가 가지는 id 속성 값과 일치시키는 방법
		<label for="email">이메일</label>
		<input required id="email" maxlength="50" minlength="18" name="email" type="email">
		-id 속성 값은 하나의  HTML 문서 내에서 고유해야 함으로 각별히 유의
