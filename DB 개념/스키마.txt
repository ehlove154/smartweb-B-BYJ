스키마
-서로 관련있는 테이블들을 묶어내기 위해 사용한다. 주로 하나의 서비스에 유관한 테이블을 묶어낸다.
-SCHEMA라는 키워드는 데이터베이스(DATABASE)라는 키워드와 치환하여 사용할 수 있다.
-실질적인 데이터를 직접 가지지 않는 단순한 껍데기
-명명법: 스네이크 케이스, 단수형 혹은 복수형

-------------------------------------------------------------

생성하기
-스키마를 생성하기 위해 아래 쿼리를 실행한다.
	CREATE SCHEMA `이름`;
	    *이미 사용 중인 이름을 사용할 수 없음에 유의한다.

-------------------------------------------------------------

조회하기
-존재하는 스키마의 목록을 조회하기 위해 아래 쿼리를 실행
	SHOW SCHEMAS;

-------------------------------------------------------------

수정하기
-스키마의 이름을 변경하는 것은 불가능

-------------------------------------------------------------

삭제하기
-존재하는 스키마를 삭제하기 위해 아래의 쿼리를 실행
	DROP SCHEMA `이름`;
	   *존재하지 않는 스키마를 삭제 할 수 없음에 유의
	   *스키마를 삭제하면 이가 가지고 있는 테이블 및 테이블의 레코드가 모두 삭제되고 되돌릴 수 없음으로 유의
